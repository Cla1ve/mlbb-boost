# BoostMLBB API — документация для интеграции

API построен на **FastAPI**. Возвращает JSON. CORS открыт (можно дёргать с фронта напрямую).
У всех ответов есть поле `success: true/false`. Есть авто-документация Swagger по `/docs`.

**Base URL:** тот же домен, где уже работает ценовой API (например `https://<api-host>`).
Все пути ниже относительные к нему.

---

## 1. Отзывы (новое) — `GET /reviews`

Возвращает опубликованные отзывы для раздела «Отзывы» на сайте.
Это ровно те отзывы, что уже видны в Telegram-канале.

> Выдаются только **успешно опубликованные** отзывы — у которых есть реальный
> пост в канале. Поэтому у каждого элемента `message_link` ведёт на существующий
> пост (валидная ссылка гарантирована).

**Query-параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `limit`  | int | `50` | Сколько отзывов вернуть. Допустимо `1..100`. |
| `offset` | int | `0`  | Смещение для пагинации. |

**Пример запроса:**

```
GET /reviews?limit=20&offset=0
```

**Пример ответа:**

```json
{
  "success": true,
  "pagination": {
    "limit": 20,
    "offset": 0,
    "returned": 20,
    "total": 844,
    "has_more": true
  },
  "data": [
    {
      "id": 1431,
      "post_no": 851,
      "rating": 5,
      "text": "Все чётко, лучший бустер)",
      "boost_type": "hero",
      "boost_type_name": "На герое",
      "is_legacy": false,
      "rank_from": "Эпик IV",
      "rank_to": "Легенда I",
      "hero": null,
      "role": null,
      "duration": null,
      "booster": {
        "name": "VenomSkie.",
        "ref": "321866bf17011847"
      },
      "published_at": "2026-06-13 11:19:01",
      "created_at": "2026-06-13 11:18:52",
      "message_link": "https://t.me/Cla1ve_boost/1285",
      "channel_url": "https://t.me/Cla1ve_boost"
    }
  ]
}
```

**Поля отзыва (`data[]`):**

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | Внутренний ID отзыва (уникальный, для `key` в списках). |
| `post_no` | int \| null | Номер публикации (как «Отзыв №851» в канале). |
| `rating` | int \| null | Оценка `0..5`. |
| `text` | string \| null | Текст отзыва. Может быть `null`, если без комментария. |
| `boost_type` | string \| null | Ключ типа буста: `standard`, `role`, `hero`, `party`, `rising_login`, `rising_party`. Часто бывает `null`. |
| `boost_type_name` | string \| null | Человекочитаемое название типа буста (рус.). |
| `is_legacy` | bool | Отзыв старого образца (у таких нет рангов). |
| `rank_from` / `rank_to` | string \| null | Ранг «откуда → куда». `null` у legacy. |
| `hero` | string \| null | Герой (если буст на герое). |
| `role` | string \| null | Роль (если применимо). |
| `duration` | string \| null | Срок выполнения заказа (текстом). |
| `booster.name` | string | Ник бустера (публичный). |
| `booster.ref` | string | Анонимный стабильный ID бустера (для группировки отзывов одного бустера). **Это не Telegram ID.** |
| `published_at` | string \| null | Дата публикации (`YYYY-MM-DD HH:MM:SS`). |
| `created_at` | string | Дата создания отзыва. |
| `message_link` | string | **Прямая ссылка на оригинальный пост с отзывом в Telegram-канале** (`https://t.me/<канал>/<message_id>`). По клику открывается Telegram и сам пост. Гарантированно валидна (выдаются только опубликованные отзывы). |
| `channel_url` | string | Ссылка на канал с отзывами (для кнопки «перейти в канал»). |

**Как выгрузить все отзывы:** листать `offset`, пока `pagination.has_more === true`
(шаг = `limit`). Общее число — в `pagination.total`.

**Сортировка:** новые сверху (по `published_at`, затем `created_at`).

**Замечания для фронта:**

- Многие поля могут быть `null` — это нормально, обрабатывайте отсутствие.
- Не считайте `booster.ref` за Telegram ID — это хэш, только для группировки.

## 2. Цены (уже используется на сайте) — `GET /prices`

```
GET /prices
```

```json
{
  "success": true,
  "data": {
    "epic": { "standard": 90, "role": 95, "hero": 130, "party": 190 },
    "legend": { "standard": 95, "role": 100, "hero": 140, "party": 200 }
  }
}
```

Категории: `warrior_elite`, `master_gm`, `epic`, `legend`, `mythic`, `honor`, `glory`, `immortal`.
Цена — за звезду.

Также доступны: `GET /prices/formatted` (с названиями), `GET /ranks`,
`POST /calculate` (расчёт стоимости буста) — менять не нужно, они уже работают.

## 3. Общие правила

- **Формат ошибок:**

  ```json
  { "detail": "Error fetching reviews" }
  ```

  HTTP-код `4xx/5xx`. Детали наружу не раскрываются (только generic-сообщение).
- **Rate limit:** `/reviews` — 60 запросов/мин с IP. При превышении — `429 Too Many Requests`.
  Рекомендуется кэшировать ответ на фронте/бэке (данные и так кэшируются на сервере ~30 сек).
- **Метод:** только `GET`. API только на чтение, ничего не изменяет.
- **Авторизация:** не требуется (публичные данные).
